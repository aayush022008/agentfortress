# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |

## Reporting a Vulnerability

If you discover a security vulnerability in AgentFortress, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: aayush022008@gmail.com

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and aim to release a patch within 7 days for critical issues.

## Security Features

AgentFortress itself is a security tool. Our own security practices include:

- All audit logs are cryptographically signed with Ed25519
- No sensitive data is stored in plaintext
- API keys are hashed before storage
- Regular dependency updates via Dependabot
- All releases are tagged and signed

## Responsible Disclosure

We follow responsible disclosure principles. Researchers who report valid vulnerabilities will be credited in the release notes (with permission).
